System Overview

This system monitors checkout lanes using AI and computer vision:

  • Detects adults, children, and items
  • Estimates wait time
  • Calculates line length
  • Identifies overloaded lanes
  • Displays a customer countdown screen
  • Generates lane history & heatmaps
  • Logs all events to logs/lane_events.csv

Flow diagram:

Camera → YOLO + ByteTrack → Lane Analytics → server.py → Dashboard + Customer Screen
                                          ↓
                             lane_events.csv (history)

Camera Tips & Performance

  • Laptop webcams → unstable exposure + limited FOV + low FPS
  • 4K external webcams → stable, wide, perfect for slanted lines
  • Avoid running multiple apps that access the camera at the same time
  • YOLOv8x6 = very accurate, but your GPU must be used (CUDA ON)
  • Close Chrome tabs for highest FPS

Tip: Built-in webcams usually can't handle slanted-line geometry. They distort too much + FOV is too small. Use vertical band mode for laptop tests.

Why “Adults in Line” Might Show 0

  • Slanted line enabled but camera angle is too flat
  • Wrong near/far points in config.yaml
  • Track age too high (use min_track_age_frames: 1)
  • Lighting too dark or overexposed
  • Laptop webcam too zoomed-in
Fix: Set use_slanted_line: false until your 4K camera arrives.

Main Configuration (config.yaml)

Important keys:

  • model.path – YOLO model
  • queue_geometry.use_slanted_line – slanted vs vertical band
  • tracking.min_track_age_frames – helps stabilizing counts
  • screenshots.enabled – saves images on big alerts
  • lanes – camera sources (“0”, “1”, RTSP, etc.)

Restart system after editing the config.

Slanted-Line Calibration

Use this ONLY when your 4K lane camera is installed.

  • Run python calibrate_queue.py
  • Click near the cashier → NEAR point
  • Click far down the aisle → FAR point
  • Press s to save
  • System automatically updates config.yaml
Do NOT use slanted-line mode on laptop webcams — adult count will break.

Shift Heatmap (Full-Day Load)

Heatmaps are built from logs/lane_events.csv using timestamps.

  • Each lane update is logged with time, customers, wait, items
  • The dashboard heatmap uses 24-hour bins
  • Shows peak load by hour
  • Great for staffing decisions
  • Automatically updates through the new API: /metrics/heatmap

Tip: the system automatically handles multiple lanes.

Troubleshooting

  • No camera feed? Close Zoom/Discord/Camera app → restart app → ensure webcam access allowed.
  • FPS too low? Reduce imgsz to 720 or switch to yolov8l.pt.
  • 0 adults? Set use_slanted_line: false.
  • Camera error -1072873821 Windows MSMF conflict → another app is using the camera.
  • Alerts not showing? Check lane_events.csv is being written.
  • Dashboard empty? Ensure backend is running: python server.py
